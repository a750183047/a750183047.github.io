html学习笔记之css

HTML学习笔记 css

什么是css？
css，Cascading Style Sheet 层叠样式表，用来给HTML页面标签添加各种样式。

为什么用CSS

HTML的缺陷：

  1. 不能够适应多种设备
  2. 要求浏览器必须智能化足够庞大
  3. 数据和显示没有分开
  4. 功能不够强大

css的优点：

1.使数据和显示分开
2.降低网络流量
3.使整个网站视觉效果一致
4.使开发效率提高了

css基本选择器

  • 标签选择器：指的就是选择器的名字代表HTML页面上的标签

  • 类选择器：规定用.来定义，优点是灵活.one{background-color:#ff0099; }

  • ID选择器：规定用#来定义，#one{cursor:hand; }区别：标签选择器针对的是页面上的一类标签，类选择器可以供多种标签使用，id选择器是只供特定的标签使用的，id是这个标签在此页面上唯一的标识
  • 通用选择器：用*来定义，代表页面上的所有标签

HTML学习笔记 表单

表单是HTML中用来提交数据的标签，里面有输入框，单选框，多选矿，字符框等多种控件。form是表单的标签。

  • <input type="text" value="请输入姓名" size="50" readonly disabled> 普通字符输入框，type规定了输入的类型，value则是输入框内的内容，如果这里有内容，则显示在输入框中。size是输入框能够输入的字符长度，readonly，只读，输入框无法输入字符，disable，输入框无法被选中。
  • <input type="password" value="密码" size="50"> type为password，输入类型为密码，则输入的字符会被·替换。
  • <input type="hidden" value="隐藏框" size="50"> type为hidden，输入框隐藏，用来给服务器发送一些数据而不对用户可见。 
  • <input type="radio" name="sex" value="man">
    男
    <input type="radio" name="sex" value="woman">
    女

单选按钮，type为radio，通过name设置相同的值规定为一组，达到单选的效果。

Android BroadcastReceiver复习

Android BroadcastReceiver复习

BrodacastReceiver，安卓四大组件之一，可见其重要性，所以学会运用它还是很重要的。
广播接收者，顾名思义，就是接收广播的，那什么是广播呢？这里的广播可不是调频广播，这里的广播其实是。。。好吧，我们来百度一下。

Android广播机制指的是，在一个应用程序运行的时候可以自定义一个消息类型，让相应的接收器去处理这个消息或者是系统消息，比如来电话了、来短信了、手机没电了等等系统发送的消息。系统发送的消息也可以通过广播的方式通知给应用程序，这样子就避免了新开一个Thread去监听系统或其他应用发送过来的消息的状态。

广播的概念其实还是很好理解的，就是说如果你的应用想要知道手机什么时候来电话了，只需要设置一个来电的广播接收器就可以了，如果来电话了，系统就会发送出一个广播，通知软件，这时候接收器就会收到这个广播，你的软件也就知道来电话了。
如果你的应用还有一个兄弟应用，你想在你的应用打开的时候偷偷的打开你的兄弟应用，那你就可以在软件中发送一个广播，如果你的兄弟应用接受到了这个广播，它就悄悄的启动，然后手机就变卡了。。。。

Android Activity启动模式

Android Activity的启动模式

在编写安卓程序的过程中，想必大家对Activity的生命周期都很熟悉了吧（不熟悉？这里有），因为很多操作都需要结合Activity的生命周期去执行，但是Activity还有一个启动模式，平时练的就很少了，感觉什么启动模式都是差不多的，没有什么区别。
但是这毕竟只是在练习，如果真的做项目了，这些细节问题很可能造成一些奇怪的Bug，所以还是要复习一下活动的启动模式，争取到遇到相关问题的时候不至于什么都不知道从而浪费更多的时间。

standard 模式

默认的启动模式。每当启动一个新的活动，他就会在返回栈中入栈，并处于栈顶的位置，对于使用该模式的活动，系统不会在乎这个活动是否存在返回栈中，每次启动活动都会创建一个新的实例。

看图片就能很快理解这种启动模式，每次启动一个活动都会在返回栈中创建一个该活动的实例，有时候我们从一个活动A中跳转到另一个活动B，又从活动B中跳回活动A（跳转是创建该活动，比如用Intent，startActivity，而不是用返回键），这时候，返回栈中就ABA这三个实例了，如果这是我们按返回键，会发现第一次从A返回到了B，再按一次返回键，又回到了A，这里的两个A并不是同一个A，这就是standard启动模式的特点。简单粗暴，什么都不管，你打开一个活动我就给你创建一个实例，如果一直在创建活动，总会把内存沾满导致死机的。

Java基础复习-接口

Java基础复习 Interface

接口，是Java中非常重要的抽象类型，接口的关键字 Interface，在使用的时候可以只定义方法体而不需要具体的实现。就像c++中的虚函数一样，一个类可以实现多个接口，从而解决了Java只有单继承的不足。

上面是接口的介绍，刚刚学Java的时候，对于接口的理解还不是很深刻，也不太明白接口的作用。但是在学习安卓的过程中，发现接口运用的非常频繁，所以必须要搞明白接口的使用，才能更好的理解安卓代码。这也算对Java基础不足的显示吧。

先看看接口的特点吧：

接口与类相似点：

  • 一个接口可以有多个方法
  • 接口文件保存在Java结尾的文件中，文件名使用接口名（如 Abc.java 中 有 public interface Abc{} 这个接口）

接口与类的区别：

  • 接口不能用于实例化对象 （不能 Abc abc = new Abc()）
  • 接口没有构造方法
  • 接口中所有的方法必须是抽象方法（什么是抽象方法？ 如 public void abcD();）
  • 接口中不能包含成员变量，除了static和final变量
  • 接口不能被类继承，而是要被类实现（public class A implement Abc{}）
  • 接口支持多继承（public class A implement Abc，Def,Ghi{}）
