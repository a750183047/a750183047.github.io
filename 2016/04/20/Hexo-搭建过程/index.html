本文参照 Zippera’s blog 的系列教程，加上自己操作时遇到的问题，总结的文章。

什么是hexo

hexo是一个机遇Node.js的静态博客程序，可以方便的生成静态网页托管在github和Heroku上。作者是来自台湾的@tommy315。

hexo：快速、简单且功能强大的 Node.js 博客框架。
A fast, simple & powerful blog framework, powered by Node.js.

好吧，上面的都是介绍，台湾的大神在大学期间就自己写出了这么牛的东西，真是佩服的五体投地，再看看我，还在这里苦逼的学着各种东西，惭愧啊。

吐槽完毕，毕竟大神也不是天生就会的，人家用自己的努力换来了现在的成就，我们也不能就这样沉沦下去，还是得好好的丰富自己的技能才行呢，怎么办才好呢？

虽然自己写的代码还算是不少，但是写过去的东西就忘记了，这个不是好的现象，为了避免这个问题，我决定还是把自己在学习过程中的知识点记录下来，这样不仅能增加印象，也能给自己留下一些东西。

开始了

安装Git

Hexo是基于git的工具，所以首先需要配置git环境，怎么配置呢？好说，直接下载安装
git 。

安装Node.js

在Windows下安装它很简单，只需要下载安装文件安装就好了。

安装hexo

利用nmp命令即可安装。（在任意位置点击右键，选择 git bush）

npm install -g hexo

tips

hexo现在支持更加简单的命令格式了，比如：

hexo g == hexo generate
hexo d == hexo deploy
hexo s == hexo server
hexo n == hexo new
